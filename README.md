# Inter Fonts — Nextcloud App

Replaces every Nextcloud interface font with **Inter**, a screen-optimised
variable typeface by Rasmus Andersson. All font files are embedded directly
inside the app — no CDN, no Google Fonts, no external requests of any kind.

## Features

| Feature | Details |
|---|---|
| Fully self-hosted | Font files live at `fonts/` inside the app |
| Variable font | One WOFF2 per axis covers weights 100–900 |
| Full UI coverage | Injected on every page via `BeforeTemplateRenderedEvent` |
| Sub-directory safe | Font URLs generated at runtime via `IURLGenerator` |
| Deep integration | Overrides Nextcloud's `--font-face` CSS variable |
| Auto-updated | GitHub Actions bumps the app version when Inter releases a new version |

## Requirements

| Dependency | Version |
|---|---|
| PHP | &ge; 8.2 |
| Nextcloud | 27 – 32 |

## Installation

Always install from the **latest release** — it includes the bundled font files.
Cloning or downloading the repository source directly will not include the
WOFF2 files (they are added by the automated release workflow).

```bash
# 1. Download the latest release archive
curl -fsSL \
  "$(curl -fsSL https://api.github.com/repos/solracsf/nc-interfonts/releases/latest \
    | grep '"browser_download_url"' | grep '\.tar\.gz' | cut -d '"' -f4)" \
  -o /tmp/interfonts.tar.gz

# -- or download the ZIP --
curl -fsSL \
  "$(curl -fsSL https://api.github.com/repos/solracsf/nc-interfonts/releases/latest \
    | grep '"zipball_url"' | cut -d '"' -f4)" \
  -o /tmp/interfonts.zip
```

You can also grab a specific version by tag:

```bash
# Replace v1.2.0 with the desired tag from the Releases page
curl -fsSL \
  https://github.com/solracsf/nc-interfonts/archive/refs/tags/v1.2.0.tar.gz \
  -o /tmp/interfonts.tar.gz
```

Then extract and enable:

```bash
# 2. Extract into the Nextcloud apps directory
tar -xzf /tmp/interfonts.tar.gz -C /var/www/nextcloud/apps/

# The extracted folder may include the tag in its name — rename it
mv /var/www/nextcloud/apps/nc-interfonts-* /var/www/nextcloud/apps/interfonts 2>/dev/null || true

# 3. Set correct ownership
chown -R www-data:www-data /var/www/nextcloud/apps/interfonts/

# 4. Enable via OCC
sudo -u www-data php /var/www/nextcloud/occ app:enable interfonts
```

> **Note:** The latest release tag and direct download links are always
> available on the [Releases page](https://github.com/solracsf/nc-interfonts/releases/latest).

## Upgrading

```bash
# Disable the running app first
sudo -u www-data php /var/www/nextcloud/occ app:disable interfonts

# Download and extract the new release (same steps as Installation above)
# then re-enable
sudo -u www-data php /var/www/nextcloud/occ app:enable interfonts
```

## Font files

The two Inter variable WOFF2 files are bundled in every release under `fonts/`:

| File | Description |
|---|---|
| `fonts/Inter.var.woff2` | Roman variable font (wght 100–900) |
| `fonts/InterItalic.var.woff2` | Italic variable font (wght 100–900) |
| `fonts/inter-version.txt` | Upstream Inter version currently bundled |

These files are updated automatically by the GitHub Actions workflow whenever
a new Inter release is published (see [Automatic font updates](#automatic-font-updates)).

## How it works

### PHP layer

1. `Application.php` registers `BeforeTemplateRenderedListener` via the
   `IBootstrap` interface — the modern Nextcloud DI bootstrap pattern.
2. The listener is constructed by the DI container with `IURLGenerator`
   injected automatically.
3. On every page render (`BeforeTemplateRenderedEvent`), the listener:
   - Calls `Util::addStyle()` to queue `css/inter-font.css` into `<head>`.
   - Calls `Util::addHeader()` to inject an inline `<style>` block that
     exposes the correct absolute font-file URLs as CSS custom properties:
     ```css
     :root {
       --inter-font-url: '/apps/interfonts/fonts/Inter.var.woff2';
       --inter-italic-font-url: '/apps/interfonts/fonts/InterItalic.var.woff2';
       --font-face: "Inter", -apple-system, ...;
     }
     ```
   URLs are generated by `IURLGenerator::linkTo()` — correct for any
   sub-directory install (e.g. `https://cloud.example.com/nextcloud/`).

### CSS layer

`css/inter-font.css` declares the two `@font-face` rules that point to the
embedded WOFF2 files, then applies `var(--font-face)` to every relevant
Nextcloud selector:

```css
@font-face {
    font-family: "Inter";
    font-style: normal;
    font-weight: 100 900;
    font-display: swap;
    src: url(var(--inter-font-url, "../fonts/Inter.var.woff2")) format("woff2"),
         url("../fonts/Inter.var.woff2") format("woff2");
}
```

The relative path fallback (`../fonts/Inter.var.woff2`) ensures the font loads
correctly even in edge cases where the inline `:root` block has not yet been
parsed.

## Automatic font updates

The workflow at `.github/workflows/update-inter-fonts.yml` runs every Sunday
at 02:00 UTC and on manual dispatch. When a new Inter release is detected on
[rsms/inter](https://github.com/rsms/inter) it automatically:

1. Downloads `Inter-X.Y.zip` from the official release assets
2. Extracts `InterVariable.woff2` and `InterVariable-Italic.woff2`
3. Bumps the app **minor** version in `appinfo/info.xml` and `CHANGELOG.md`
4. Updates the `Bundled Inter version:` comment in `css/inter-font.css`
5. Commits all changes directly to `main`
6. Creates and pushes a Git tag (`vX.Y.0`)
7. Publishes a GitHub Release with full release notes

No pull request is opened — the release is fully automatic.

To trigger an update manually or force a specific Inter version:
1. Go to **[Actions → Update Inter Fonts → Run workflow](https://github.com/solracsf/nc-interfonts/actions/workflows/update-inter-fonts.yml)**
2. Leave `force_version` empty for the latest, or enter a tag (e.g. `v4.2`)
3. Click **Run workflow**
