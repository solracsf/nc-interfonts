## Installation

Always install from the **[latest release](https://github.com/solracsf/nc-interfonts/releases/latest)** â€”
it includes the bundled Inter font files. Cloning or downloading the repository
source directly will not include the WOFF2 files (they are committed by the
automated release workflow).

```bash
# 1. Resolve the latest release tag and download the tarball
TAG=$(curl -fsSL https://api.github.com/repos/solracsf/nc-interfonts/releases/latest \
  | grep '"tag_name"' | cut -d '"' -f4)

curl -fsSL \
  "https://github.com/solracsf/nc-interfonts/archive/refs/tags/${TAG}.tar.gz" \
  -o /tmp/interfonts.tar.gz

# 2. Extract into the Nextcloud apps directory
tar -xzf /tmp/interfonts.tar.gz -C /var/www/nextcloud/apps/

# The extracted folder is named nc-interfonts-<tag> â€” rename it
mv "/var/www/nextcloud/apps/nc-interfonts-${TAG#v}" \
   /var/www/nextcloud/apps/interfonts

# 3. Set correct ownership
chown -R www-data:www-data /var/www/nextcloud/apps/interfonts/

# 4. Enable via OCC
sudo -u www-data php /var/www/nextcloud/occ app:enable interfonts
```

To pin a specific version, replace the tag resolution step:

```bash
# Replace v1.2.0 with any tag listed on the Releases page
TAG=v1.2.0

curl -fsSL \
  "https://github.com/solracsf/nc-interfonts/archive/refs/tags/${TAG}.tar.gz" \
  -o /tmp/interfonts.tar.gz
```

> **Note:** Download links for every release are also listed on the
> [Releases page](https://github.com/solracsf/nc-interfonts/releases).

## Upgrading

```bash
# 1. Disable the running app
sudo -u www-data php /var/www/nextcloud/occ app:disable interfonts

# 2. Remove the old version
rm -rf /var/www/nextcloud/apps/interfonts

# 3. Download and install the new release (same steps as Installation above)

# 4. Re-enable
sudo -u www-data php /var/www/nextcloud/occ app:enable interfonts
```

## How it works

### PHP layer

1. `Application.php` registers `BeforeTemplateRenderedListener` via the
   `IBootstrap` interface â€” the modern Nextcloud DI bootstrap pattern.
2. The listener is constructed by the DI container with `IURLGenerator`
   injected automatically.
3. On every page render (`BeforeTemplateRenderedEvent`), the listener:
   - Calls `Util::addStyle()` to queue `css/inter-font.css` into `<head>`.
   - Calls `Util::addHeader()` to inject an inline `<style>` block that
     exposes the correct absolute font-file URLs as CSS custom properties:
     ```css
     :root {
       --inter-font-url: '/apps/interfonts/fonts/Inter.var.woff2';
       --inter-italic-font-url: '/apps/interfonts/fonts/InterItalic.var.woff2';
       --font-face: "Inter", -apple-system, ...;
     }
     ```
   URLs are generated by `IURLGenerator::linkTo()` â€” correct for any
   sub-directory install (e.g. `https://cloud.example.com/nextcloud/`).

### CSS layer

`css/inter-font.css` declares the two `@font-face` rules that point to the
embedded WOFF2 files, then applies `var(--font-face)` to every relevant
Nextcloud selector:

```css
@font-face {
    font-family: "Inter";
    font-style: normal;
    font-weight: 100 900;
    font-display: swap;
    src: url(var(--inter-font-url, "../fonts/Inter.var.woff2")) format("woff2"),
         url("../fonts/Inter.var.woff2") format("woff2");
}
```

The relative path fallback (`../fonts/Inter.var.woff2`) ensures the font loads
correctly even in edge cases where the inline `:root` block has not yet been
parsed.

## Automatic font updates

The workflow at `.github/workflows/update-inter-fonts.yml` runs every Sunday
at 02:00 UTC and on manual dispatch. When a new Inter release is detected on
[rsms/inter](https://github.com/rsms/inter) it automatically:

1. Downloads `Inter-X.Y.zip` from the official release assets
2. Extracts `InterVariable.woff2` and `InterVariable-Italic.woff2`
3. Bumps the app **minor** version in `appinfo/info.xml` and `CHANGELOG.md`
4. Updates the `Bundled Inter version:` comment in `css/inter-font.css`
5. Commits all changes directly to `main`
6. Creates and pushes a Git tag (`vX.Y.0`)
7. Publishes a GitHub Release with full release notes

No pull request is opened â€” the release is fully automatic.

To trigger an update manually or force a specific Inter version:
1. Go to **[Actions â†’ Update Inter Fonts â†’ Run workflow](https://github.com/solracsf/nc-interfonts/actions/workflows/update-inter-fonts.yml)**
2. Leave `force_version` empty for the latest, or enter a tag (e.g. `v4.2`)
3. Click **Run workflow**
